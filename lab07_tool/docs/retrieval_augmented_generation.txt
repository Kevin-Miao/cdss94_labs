Retrieval-Augmented Generation

Retrieval-Augmented Generation (RAG) is a technique that combines information retrieval with text generation to improve the factual accuracy and knowledge coverage of language models. The approach was introduced by Lewis et al. (2020) and has become a foundational pattern in modern AI systems.

In a RAG system, when a user asks a question, the system first retrieves relevant documents from a knowledge base, then provides those documents as context to the language model along with the original question. This allows the model to generate answers grounded in specific, up-to-date information rather than relying solely on knowledge encoded during training.

The retrieval step typically uses dense retrieval, where both the query and documents are encoded as dense vectors (embeddings) using models like sentence-transformers. The system finds the most relevant documents by computing cosine similarity or inner product between the query embedding and document embeddings stored in a vector database such as FAISS, Pinecone, or Weaviate.

Key advantages of RAG include: reduced hallucination since the model can reference specific source material; easy knowledge updates by modifying the document store without retraining; attribution and transparency since retrieved passages can be shown to users; and cost efficiency compared to fine-tuning on domain-specific data.

Common challenges include retrieval quality (the system is only as good as the documents it retrieves), chunk size selection (too small loses context, too large dilutes relevance), and the model's ability to synthesize information from multiple retrieved passages. Advanced RAG systems use techniques like re-ranking, query expansion, and hybrid search (combining dense and sparse retrieval) to improve performance.
